# Deploying CollegeManagement on Render

This repo is ready for Render using the `render.yaml` Blueprint.

## 1. Push to GitHub

```bash
git init
git add .
git commit -m "Prepare Render deployment"
git branch -M main
git remote add origin https://github.com/YOUR_USERNAME/YOUR_REPO.git
git push -u origin main
```

If the repo already exists locally, only run:

```bash
git add .
git commit -m "Prepare Render deployment"
git push
```

## 2. Create Render Blueprint

1. Open Render.
2. Choose **New +**.
3. Choose **Blueprint**.
4. Connect your GitHub repository.
5. Select this repo.
6. Render will detect `render.yaml`.

It will create:

- `college-management-backend`
- `college-management-frontend`

## 3. Backend Environment Variables

In Render, set these for the backend service:

```env
MONGO_URI=mongodb+srv://USERNAME:PASSWORD@cluster0.xxxxx.mongodb.net/
MONGO_DB_NAME=college_management
CLIENT_URL=https://college-management-frontend.onrender.com
```

`JWT_SECRET` is generated by Render from `render.yaml`.

## 4. Frontend Environment Variable

In Render, set this for the frontend static site:

```env
VITE_API_URL=https://college-management-backend.onrender.com/api
```

If Render gives different service URLs, update `CLIENT_URL` and `VITE_API_URL` to match the actual URLs.

## 5. Seed Demo Data on Render

After the backend deploys, open the backend service shell on Render and run:

```bash
npm run seed
```

To only restore the admin account without clearing demo data:

```bash
npm run ensure-admin
```

Admin login:

```text
jennifer.davis@admin.edu

```

## 6. SPA Routing

The frontend static site has a rewrite rule in `render.yaml`:

```yaml
routes:
  - type: rewrite
    source: /*
    destination: /index.html
```

This keeps React Router pages working after refresh.
